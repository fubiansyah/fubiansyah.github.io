McDelivery In-game Apps Study Case

Client: McDonalds Singapore

Activity Time : 3 - 30 September 2020

Who doesn’t love the Discount promotion?? Every single person in the world is looking for a discount promotion unless they are crazy rich enough to ignore any discount.

Mcdonald’s has an order App’s (McDelivery App’s) which allows people to order the food via their app’s. The App’s also offer so many discount promotions. But the marketing team thinks that it will be great to give some fun challenges to get the discount promotion code.

Instead of just letting the user use the “discount” promotion by clicking the promo code, the marketing team tries to give the best experience while using the Apps to get the discount promotion. “The In App’s Game” ~ the way to make user feel the fun experience inside the App’s

OBJECTIVE

Create a Minions-themed game in MCD app to increase customer engagement & grow app downloads

asdas

Challenges

The Marketing Team wants to create a simple Game with "Fun" Minion Experience. Small interaction but getting “Fun” experience it's their main purpose. They don't want to make users feel the game is too hard because they just want to get a discount promotion.

They want to seperate the Game into 2 Phase

Phase 1

Game Commences to Generate Campaign Hype & Excitement

Phase 2

More & Bigger Prizes with Cheese Sauce Bottle Launch

Game Mechanics

User Journey

Game Concept

App Homepage

While opening the app, the user sees minions taking over the entire MCD App page. Top 2 tiles are also reserved for game play.

It will get attraction from users so they will tap on either tile to play the game.

Audio Select Page

After clicking on the tile, user is directed to the loading page where they can choose to play the game with or without audio.

Loading page begins already with audio playing to give user a taste of the full visual + audio experience from the beginning.

Game Action

1. When game page is loaded, gift box shakes for a quick few seconds to get user to tap it.

2. User taps the gift box and Minion flies in with visual effects. Either Kevin or Stuart will appear (at random) every time the User taps the screen.

3. After 5 punches/kicks, the gift box shakes & opens to reveal the game prize.

Game Preview

Deals Reveal and Failed Page

At the end of game journey, it will showing reveal Deal Page (discount promotion page) here the look of the reveal page

Users that already claim the deal within the day are unable to get the reveal deal anymore. So they will direct to the failed Page. We have prepare 2 failed Page

  • Failed page for user who try to reveal the deal within the day
  • Failed page when the deal reach the limit of number

Conclusion

We got positive feedback after the game launch. However we also got another input from the user. Most users didn't know that they only can play a game once within a day. So they are always getting into failed pages. The user really often misses the terms and condition page.

So we decided to put a popup after the user clicks the title, before they start playing a game. So it will make them clear regarding game rule before they play the game.
